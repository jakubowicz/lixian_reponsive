Architecture

Recencer les Produits de l'Entreprise

Lixian repose sur une modélisation forte de l'entreprise et de ses activités: L'entreprise dispose d'un patrimoine fait de produit et/ ou de service, elle crée de la richesse en le faisant évoluer grace à ses employés qui réalisent des projets pour le compte de leurs clients. Il est donc indispensable de bien décrire l'architecture des produits et services de l'entreprise grace à une cartographie détaillée.

Produits

Produits et Services, ...

Les Produits (et Services) représentent le Patrimoine de l'Entreprise. Ce sont des produits financiers pour les banques, des applications informatiques pour les Direction des systèmes d'Information, des formations pour les écoles, ........... Leur description détaillée (Fonctionnelle, Techniques, Adhérences, Données échangées, ... ) est essentielle pour créer de la valeur

Modules

Ihms, Regles de Gestion, ..

Chaque Produit ou service peut être décrit fonctionnellement en le découpant en modules. Si l'on prend l'exemple d'une application Informatique, les modules sont les Ihm (écrans de saisie/ consultation), les règles de gestion (les algorythmes de calculs), les interfaces lui permettant de communiquer avec son éco-système.

Interfaces

Adherences, Echanges...

Chaque Produit ou service Peut être adhérent à un autre Produit/ service de l'entreprise ou d'une entreprise externe. Cette adhérence peut être une interface d'échange de données (par exemple récupérer les informations météo), un lien d'hébergement (entre une application et un serveur).

Donnees

Objets Metiers, Techniques, ...

Chaque Interface de type Synchrone ou Asynchrone entre deux Produits pemet d'échanger des données qu'on pourra appeler objets métiers ou Techniques. Les Objets métiers sont une description fonctionnelle de la donnée, les Objets techniques sont des représentations physiques (Bases de données, tables, Web services, .....)

Technologie

OS, BD, Middlewares, ...

Il est important de décire quelles sont les technologies nécessaires pour l'élaboration d'un produit ou d'un service. En effet, si une technologie venait a devenir obsolète, il serait important de pouvoir anticiper cette rupture en listant par exemple tous les produits utilisant cette technologie. Ces technologies peuvent aussi être croisées avec les compétences des collaborateurs de l'entreprise afin d'identifier les meilleures ressources à mettre sur un projet.

Documentation

Expressions de besoins, Devis, ...

Lixian permet d'attacher aux fiches descriptives des produits et services les documentations essentielles permettant de les décrire, de les faire évoluer et de les maintenir.
